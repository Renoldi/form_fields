"Refactor kode agar lebih modular dan terorganisasi dengan langkah bertahap berikut:

1. Pisahkan setiap fungsi, helper, dan bagian UI utama ke file terpisah sesuai tanggung jawabnya (misal: input handling, validation, formatting, widgets, dsb).
2. Pastikan setiap file hanya berisi satu tanggung jawab utama (single responsibility).
3. Gunakan ekspor dan impor yang jelas, serta dokumentasi singkat di setiap file/fungsi.
4. Setelah setiap tahap refactor, pastikan seluruh kode tetap berjalan tanpa error dan logic benar-benar berpindah tanpa kehilangan fungsionalitas. Validasi hasil sebelum lanjut ke tahap berikutnya.
5. Terapkan best practice Dart/Flutter untuk penamaan, struktur folder, dan reusability (misal: gunakan extension, mixin, atau class helper jika perlu).
6. Jangan ada duplikasi kode, dan pastikan semua dependensi antar file jelas.
7. Setelah seluruh refactor selesai, pastikan kode mudah dipahami, mudah diuji, dan mudah dikembangkan lebih lanjut."
