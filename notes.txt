

PROMPT REFAKTOR MODULAR (mulai dari yang mudah):


Refactor kode agar lebih modular dan terorganisasi dengan langkah bertahap berikut (semua logic harus terpisah dari UI):

1. Mulai dari yang paling mudah: identifikasi dan pisahkan fungsi/helper yang mudah dipindah ke file utilitas terpisah.
2. Pisahkan setiap fungsi, helper, dan bagian UI utama ke file terpisah sesuai tanggung jawabnya (misal: input handling, validation, formatting, widgets, dsb).
3. Pastikan setiap file hanya berisi satu tanggung jawab utama (single responsibility).
4. Semua logic (proses, validasi, helper, utilitas) harus terpisah dari UI/widget. UI hanya untuk tampilan dan interaksi.
5. Gunakan ekspor dan impor yang jelas, serta dokumentasi singkat di setiap file/fungsi.
6. Setelah setiap tahap refactor, pastikan seluruh kode tetap berjalan tanpa error dan logic benar-benar berpindah tanpa kehilangan fungsionalitas. Validasi hasil sebelum lanjut ke tahap berikutnya. Setelah satu proses refaktor selesai dan sudah berjalan baik, baru lanjut ke proses berikutnya.
7. Terapkan best practice Dart/Flutter untuk penamaan, struktur folder, dan reusability (misal: gunakan extension, mixin, atau class helper jika perlu).
8. Jangan ada duplikasi kode, dan pastikan semua dependensi antar file jelas.
9. Setelah seluruh refactor selesai, pastikan kode mudah dipahami, mudah diuji, dan mudah dikembangkan lebih lanjut.
